Q1. What are animations in Flutter?
ans) Animations in flutter are visual transitions that make UI changes smooth instead of instant.
 They improve user experience by creating motion, feedback, and interactive effects.  Example:
 *Button scaling       *Page transitions         *Loading indications         *Fade effect.

Q2. Why are animations important in mobile apps?
ans) Animations improve user experience by making apps feel smoother, more interactive and 
 visualling appealing. They also help user understand state changes and navigation flow.
 Example: *Loading animations -> tells user app is working.
 *Page transitions -> shows navigation transitions.
 *Button transitions -> gives interation feedback.

Q3) Difference between implicit and explicit animations?
ans) Implicit Animation: Flutter manage animation automatically. It's easier to implement.
     -------- ---------- Less coding, Limited Control and Example: AnimatedContainer.
ii. Explicit Animation: Developer controls animations manually. More powerful and customizable.
    -------- ---------- More code, Full Control and Example: AnimationController.
BEST INTERVIEW LINE: Implicit animations are simple and automatically handle by flutter, 
---- --------- ----- while explicit animation provide fine grained control using 
                  AnimationController.

Q4) which animation type is easier to use?
ans) Implicit animations: Because Flutter internally handles:
     -------- ----------- *animation controller      *interpolation      *timing
Example: *AnimationContainer    *AnimatedOpacity.

Q5) which animation type gives more control?
ans) Explicit animations: Because developer can control:
*duration    *direction    *curves    *repetition     *synchronization.
using: *AnimationController     *Tween    *CurvedAnimation.

                     IMPILCIT ANIMATION QUESTION
                     -------- --------- --------
Q1) what are implicit animations?
ans) This animations where flutter automatically animates property changes without 
requiring manual animation control. Example:
*AnimatedContainer      *AnimatedOpacity       *AnimatedAlign.

Q2) How does AnimatedContainer work internally?
ans) AnimatedContainer internally uses AnimationController and Tween animations 
managed automatically by flutter. when properties changes:
1. Flutter detects old and new values.          2. creates tweens
3. Animates between them smoothly.

Q3) Difference between Container and AnimatedContainer?
ans) Is Container changes instanlty with no animation support and very lightweight.
While AnimatedContainer will changes smoothly with animation with Built-in 
Animation support But it's slightly heavier.
Example: Container Code:             width = 200 instantly
-------- AnimatedContainer Code:      width animates from 100 → 200.

Q4) what happens when value change in implicit animations?
ans) FLutter automatically starts an animation from old value to the new value 
using internally managed tweens and animation controllers.

Q5) when should you use implicit animations?
ans) Use when: *simple UI transitions are needed        *minimal control is enough.
*quick animations are required.
Examples:  *button resize        *fade effect         *alignment change

                           EXPLICIT ANIMATION QUESTIONS
                           -------- --------- ---------
Q11) what are explicit animations?
ans) explicit animations are animations where the developer manually controls the 
animation using AnimationController.
They are used for:  *advanced animations   *synchronized animations     *repeating animations.
*custom motion effect.

Q12) Why do we need AnimationController?
ans) AnimationController controls the animation lifecycle such as starting, stoping, reversing,
repeating and tracking animation progress.
It provides values from code:                    0.0 → 1.0       used to animate widgets.   

Q13) what is tween?
ans) A tween defines the range between the start and end values of an animation.
Example:        Tween<double>(begin: 0, end: 100)
Flutter interpolates values between them.

Q14) what does curvedAnimation do?
ans) Curved Animations modifies the animation speed pattern to make moment feel natural.
Example: *easeIn    *bounceOut     *elasticIn
without curves:    *motion is linear and robotic.

Q15) what is AnimatedBuilder?
ans) AnimatedBuilder is a widget that rebuilds only the animated portion of the UI when 
animation values changes.

Q16) Why is AnimatedBuilder more optimized?
ans) Animated Builder minimizes unnecessary rebuilds by rebuilding only the animation dependent 
widget instead of the entire widget tree.

                          VSYNC QUESTIONS
                          ----- ---------
Q1) what is vsync?
ans) Vsync synchronizes animations with device screen refresh rate to avoid unnecessary frame
rendering.

Q2) Why do we use 'ThickProviderStateMixin'?
ans) ThickProviderStateMixin provides vsync support to AnimationController.
Example: AnimationController(  vsync: this,  )

Q3) What problem does vsync solve?
ans) Vsync prevents offscreen or inactive animations from consuming CPU and GPU
 resources unnecessarily.

Q4) What happens if we don’t use vsync?
ans) Animations may continue consuming resources even when not visible, leading to battery 
 drain and performance issues.

                         PERFORMANCE QUESTION
                         ----------- --------
Q1) why can animations cause lag? 
ans) Animations can cause lag due to excessive widget rebuilds, heavy painting, 
expensive layout or GPU overload. 

Q2) How do you optimize flutter animations?
ans) Animations can be optimized by minimizing rebuilds, using Animation Builder,
avoiding heavy UI work during frames and using vsync properly.

Q3) Why should AnimationController.dispose() be called?
ans) 'dispose()' releases animations resources and stops thickers 
to prevent momory leaks.

Q4) How does AnimatedBuilder reduce rebuilds?
ans) AnimatedBuilder rebuilds only the animated widget subtree instead of rebuilding
the whole screen.

Q5) Why should we avoid rebuilding the whole widget tree during animation?
ans) Frequent full rebuild increase CPU work and can cause dropped frames and junk.

                       RENDERING AND ENGINE QUESTIONS
                       --------- --- ------ ---------
Q1) Why are Flutter Animations Smooth?
ans) Flutter animations are smooth because flutter directly controls rendering using the skia
graphics engine and renders at high frame rates.

Q2) What role does the Skia engine play?
ans) skia is flutter's rendering engine responsible for drawing UI, animations, text,
graphics efficiently.

Q3) How many FPS does flutter target?
ans) Flutter target 60 FPS normally But 120 FPS on supported devices.

Q4) Difference between UI thread and Raster thread?
ans) UI thread builds Widgets/layouts And It handles Dart code with CPU focused working.
While Raster thread draws pixels on screen, handles GPu rendering with GPU focused working.
BEST LINE:
The UI thread prepares frames, while the raster thread renders them onto the screen.
