         DIFF B/W CANARY DEPLYMENT & STAGED ROLLOUT
Core Difference:
 Staged Rollout → Gradually release a new version 
                  to users in phases.
 Canary Deployment → Release a new version to a small 
                   subset of users first to test it in 
                   real-world conditions.
=> In short:
  Canary = a type of staged rollout with monitoring 
           + validation focus.
 1. STAGED ROLLOUT:
* You release your app/update in percentage.      5% -> 20% -> 50% -> 100%
* Used in:   i. Mobile apps (Play Store/App Store).      ii. feature release
 # Goal:     *reduce risk      *control exposure.
 # Example:   You push a Flutter app update:
   * Day 1: 10% users    * Day 2: 50% users      *Day 3: 100% of users.

 2. Canary Development:
*You release to a small controlled of Group first.
*Monitor:  *Errors.   *Performance    *Crashes.
*Then decide:   *Roll forward Or rollback.
*Goal: *Test in productiom safelly.
*Example:  i. 1% of users get new API version.
 ii. Monitor logs, creahes.        iii. if stable  -> increase traffic.

           #INTERVIEW TRAP
i.If interviewer asks:
"Are they the same?"  ans) Canary Deveopment is a specified form of staged rollout
   focused on validating a release with small user group before wider exposure.

           #PRO LEVEL ANSWER:
Staged rollout is a general strategy where a release is gradually to users in phases,
while canary development is a more controlled approach where a small subset of users 
receives a new version first, and the system is actively monitor before scaling rollout.
