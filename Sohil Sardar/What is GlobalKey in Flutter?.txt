

Q1. What is GlobalKey in Flutter?
ans) Global key is a special type of key in flutter that gives a widget a unique identity
across the entire widget. Unlike normal keys which work only among siblings, Globalkey
can identify the widget anywhere in the tree. It also allows us to access the 
widget's state object and BuildContext.

Q2) Why do we need Globalkey?
ans) we use Global key when we need direct access to widget's internal state or context 
from outside that widget. Common Use Cases:
*Form Validation.          *Accessing State method         *Getting Widget position
*controlling Scaffold/Snackbar.
Example:                 formKey.currentState!.validate();
Here we directly access the form state.

Q3) Difference between Key, ValueKey, ObjectKey, UniqueKey, GlobalKey?
ans) i. Key --> helps flutter identify widgets during rebuilds and helps preserve
widget state when the widget tree changes.

ii. ValueKey --> Example:  ValueKey(1)
ValueKey identifies a widget using a value. if two widgets have the same value, 
Flutter consider them equally. Example:
List items:   ListTile(  key: ValueKey(user.id)  )

iii. ObjectKey --> Example:  ObjectKey(user)
ObjectKey identifies widgets using object equality. It compares the actual object instance.
Used When Object themselves represent identity.

iv. UniqueKey --> Example: UniqueKey()
UniqueKey creates a completely new identity every time. No two UniqueKeys are equal.
Used when we want flutter to treat widgets are completely different.

v. GlobalKey --> Provides identity across the entire widget tree gives 
 access to State and Context.

Q4) How does Flutter identify widgets internally?
ans) Flutter identify widgets using three things: Widget type, Key, and position
in the widget tree. flutter compares old widgets with new widget to decide whether to 
update and existing Element or create a new one.

Q5) Where Is Globalkey store widget identity?
ans) GlobalKey identity is maintained inside Flutter's framework through a global key registry.
Flutter keeps a mapping between GlobalKey and its associated Element.
Concept:
 Globalkey - -> Element - -> state. 

Q6) Difference between Widget, Element, RenderObject?
ans) Widget --> It's an immutable configuration that describes how the UI should look.
Example: Text("Hello")                Widget is build Frequently.
Element --> Is the bridge between widget and RenderObject. It manage the widget lifecycle
and store the actual location of the widget in the tree.
                                                  Element is where state is attached.
Render Object --> It handles the actual layout and painting of UI on screen. Example:
*Size Calculation          *Position           *Drawing pixels.

Q7) Why does GlobalKey work with Element tree instead of Widget tree?
ans) Widgets are immutable and recreated during rebuilds,so they cannot store identity
permanently. Elements are long lived objects that manage widget lifecycle, therefore
Gloabelkey attaches identity to elements.

                     CURRENT STATE VS CURRENT CONTEXT QUESTIONS
                     ------- ----- -- ------- ------- ---------
Q8) What is currentState in GlobalState?
ans) currentState gives access to the State object of the Stateful Widget associated with that Global key.
Example: final formKey = GlobalKey<FormState>();                         formKey.currentState!.validate();
Here: GlobalKey - -> FormState

Q9) What is currentContext?
ans) currentContext give access to the BuildContext of the widget associated with the GlobalKey.
Uses: *Final widget position       *MediaQuery access         *Showing Snackbar.
Example:               formKey.currentContext

                       FORM VALIDATION QUESTIONS:
                       ---- ---------- ----------
Q10) Why do we use GlobalKey as Form?
ans) Flutter form widget internally maintains a FormState Object. GlobalKey allows us to access that
FormState from Outside and call methods like validate() and save().
Example: Form( key: formKey,  child: Column(  children:[ ] ) )
         Then:        formKey.currentState!.validate(); 

Q11) Why not use normal state management for form validation?
ans) State management manages application data, but Form validation is the responsibility of form state.
Using Global key here is simpler because we only need direct access to one widget internal behavior.

Q12) What happens when widget with Global key moves?
ans) Flutter recognizes the GlobalKey and finds the existing Element instead of creating a new one. 
Because the Element is reused, the widget keeps its previous State.
Example: Moving:

Q13) Why does global key preserve widget state?
ans) Because Flutter uses the GlobalKey to locate the existing Element. Since the Element containing 
the State is reused, the State object remains alive.

                       GLOBALKEY VS STATE MANAGEMENT
                       --------- -- ----- ----------
Q14) Why not use GlobalKey instead of Provider/GetX/Bloc/Riverpod?
ans) GLobalkey is not designed for managing application state. It only provide direct access to specific 
widget.State management solutions handle sharing and updating data across multiple widgets.

Q15) Can global key replace state management?
ans) No, Global key works for widget level control, while state management works for application level 
data flow.

Q16) When use GlobalKey vs State Management?
ans) GlobalKey: *Form Validation     *Access Widget Methods
                *Scaffold Control
State Management: *User Data      *Authentication      *Cart       *Theme      *Api state

                     PERFORMANCE QUESTIONS
                     ----------- ---------
Q17) Why can too many GlobalKeys hurt performance?
ans) Because Flutter has to maintain a global registry and track every Global key across the
entire widget tree. Excessive usage increases lookup work and widget coupling.

Q18) Why avoid global key for normal communication?
ans) Because it creates direct dependency between widgets. Flutter architecture prefer passing data 
downward and callbacks upward.

                    CLEAN ARCITECTURE QUESTIONS:
                    ----- ----------- ----------
Q19) Why is global key consider imparative?
ans) Because we directly tell a widget what action to perform instead of updating data and letting 
Flutter rebuilf the UI declaratively.
Example: Imperative:      formKey.currentState.validate();
Declarative:        Update state → UI rebuilds.

Q20) Why passing callbacks is often better?
ans) Callbacks keep widget independent and reusable because the child does not need to know about the 
parent's internal implementation.

                     TRAP QUESTIONS:
                     ---- ----------
Q21) Can GlobalKey access any widget in the app?
ans) NO, it only access the widget that is currently associated with that specific Globalkey.

Q22) Should every Stateful widget need GlobalKey?
ans) No, most stateful widget do not need Globalkey. It should only be used when we need 
external access to state or context.
