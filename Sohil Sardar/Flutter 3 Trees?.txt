1. What are Flutter’s 3 trees?
ans) Flutter Internally works using 3 major trees:
1. Widget Tree             2. Element Tree           3. RenderObject Tree     
BEST INTERVIEW: Flutter Separates UI configuration, lifecycle management
---- ---------- and rendering into 3 different trees:
Widget tree, Element Tree and RenderObject tree.

2. Why does Flutter use multiple trees instead of one?
ans) Because each tree has a separate responsibiltiy.
 if everything were inside one tree:
*rendering would be expensive.       *state handling would be difficult.
*rendering optimization would suffer.
BEST INTERVIEW: Flutter uses multiple trees to separate concerns. Widgets describe UI,
---- ---------- Elements manage lifecycle and state, and Rendering object handle layout
and painting. This separation improves performance and reusability.

3. What's the purpose of each tree?
ans) Widget tree: UI configuration/blueprint.
Element tree: Connects widgets with render object and stores data.
Render Object: Handle layout and paints pixels.

4. which tree is immutable?
ans) Widget tree: Widgets are immutable configuration objects.

5. Which tree actually renders UI?
ans) RenderObject tree Because:
*calculate layout           *handle painting        *draw pixels on screen.

                  WIDGET TREE QUESTIONS:
1. Why are widgets immutable?
ans) Because immutable objects are:
*lightweight       *predictable       *easy to rebuild.       *easy to optimize
Flutter simply creates new widget object instead of modifting old ones.
BEST QUESTIONS ANSWER: Widgets are immutable because flutter rebuilds UI frequently.
---- --------- ------- creating lightweight immutable configuration is faster and safer
then mutating existing UI objects.

2. Why is rebuilding widgets considered cheap?
ans) Because widgets are only configuration objects. They:
*do not paint pixels.        *do not perform layout.       *Are lightweight dart objects.
BEST INTERVIEW ANSWERS: Rebuilding widgets are cheap because widgets are just lightweight
---- --------- -------- configuration objects, not actual rendered UI elements.

3. What happens to the Widget tree during setState()?
ans) when 'setState()' is called:
*Flutter marks the element dirty.
*build method runs again.       *new widgets are recreated.
BEST INTERVIEW ANSWER: During 'setState', Flutter recreates the artifacted widget
---- --------- ------- subtree by calling the build method again.

4. Does Flutter recreates the entire app on rebuild?
ans) No, only widgets recreate. Flutter try to reuse:
*Elements          *RenderObjects        whenever possible.
Correct Interview Answer: Flutter recreates widget during rebuilds, but 
------- --------- ------- Elements and RenderObjects are mostily reused,
so the entire app is not recreated.

                    ELEMENT TREE QUESTIONS:
1. What is Element Tree?
ans) The Element tree is a bridge between:  *Widgets       *RenderObjects
It manages:   *Widget lifecycle      *state       *widget updates
BEST INTERVIEW ANSWER: The Element tree connect immutable widgets with mutable renderObject
---- --------- ------- and manage lifecycle and state preservation.

2. what is BuildContext Internally?
ans) 'BuildContext' is actually an element object.
BEST INTERVIEW ANSWER: BuildContext is basically a reference to an element 
---- --------- ------- in the element tree.

3. Why does flutter need Elements?
ans) Because widgets are immutable and temporary.
Flutter Needs Element to:  *prevent state    *track widget positions     *reuse render Objects.
Best Interview Answer: Flutter uses Elements to maintain persistent relationships 
                       between widgets and RenderObjects across rebuilds.

4. How does Flutter Preserve state during rebuild?
ans) Because state is stored in:    *stateful Element     *State object.       Not inside Widget.
BEST INTERVIEW ANSWER: Flutter preserves state because state lives in the 
                       Element tree, not in widgets. 

5. what is reconciliation in Flutter?
ans) Reconcilination means:       *comparing old widget with new widgets.
*deciding whether Elements and RenderObjects can be reused.
BEST INTERVIEW ANSWER: Reconcilination is Flutter's process of matching old 
---- --------- ------- and new widgets to resuse existing elements and RenderObjects efficiently.

               SUPER COMMON TRAP QUESTIONS:
1. If widgets are recreated every rebuild, why doesn’t state reset?
ans) Because state is stored inside the element tree through stateful element and state objects,
     not inside objects.

               RENDER OBJECT TREE QUESTIONS:
1. what does the RenderObject tree do?
ans) It handles:  *layout  *sizing  *positioning  *painting

2. which tree handles layout?
ans) RenderObject Tree.

3. which tree repaints pixels on screen?
ans) RenderObject Tree.

4. Why are RenderObjects expensive?
ans) Because they perform: *layout calculations  *painting  *compositing.
These operations are CPU/GPU intensive.
BEST INTERVIEW ANSWERS: RenderObjects are expensive because they perform actual rendering work
                        like layout and painting.

5. what happens if layout changes?
ans) Flutter marks affected RenderObjects dirty and recalculates layout.

             LIFECYCLE AND SET STATE QUESTIONS?
1. What exactly happens internally when setState() is called?
ans) Flow: *Element marked dirty    *Scheduled for rebuild     *build() runs
*New widgets generated     *UI repaints if necessary   *Reconciliation happens
*RenderObjects update if needed
BEST INTERVIEW ANSWER: 'setState()' marks the element dirty, triggers rebuild,
---- --------- ------- performance reconciliation and updates RenderObjects only if required.
